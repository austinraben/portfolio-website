Who is Austin

I am a computer science graduate of the University of Arizona, where I also minored in business administration. My focus is networking: I earned my CCNA and work as a network technician at RabenIT, handling layer 1 and layer 2 infrastructure and helping people make sense of their technology.

I like understanding systems all the way down, from the object-oriented design of an application to the physical and data-link layers of a network. Off the clock I am laid back and easy to work with, a big sports fan (basketball, football, and baseball, which is part of what drew me to the baseball analytics work), and I run my own corner of the internet, including this site and my email at austinraben.com. When I build something, I take it all the way to running in the real world, not just working on my laptop.

Enterprise Network Automation

A two-part networking project. The first is an enterprise network designed and simulated in Cisco Packet Tracer, built from routers, core and distribution switches, and edge devices. The second is a Python automation script that connects to live Cisco devices with Netmiko, captures their running configurations into date-stamped backups every night, and flags configuration drift by comparing each day against the last. It handles timeouts and authentication failures gracefully and reports a summary of every run. Built alongside earning my CCNA certification.

  • Cisco
  • Python
  • Netmiko

Restaurant Management System

A command-line application for running a restaurant from the perspective of a manager, host, and server. It handles the full floor workflow: seating customers, assigning servers to tables, taking and modifying orders against a live menu, splitting checks, collecting tips, and applying happy hour pricing, with manager tools for hiring, sales tracking, and secure login. Built in Java with an MVC structure and object-oriented design, and written entirely by hand without GenAI. It was a four-person group project for an upper-division software engineering course. I built the menu system, implemented the Happy Hour feature using the Decorator design pattern, added input validation, and wrote a large share of the unit tests.

  • Java
  • OOP
  • MVC

Music Library System

A Java application that models the data behind a music streaming service. Users browse a music store, add albums and songs to a personal library, build and manage playlists, rate songs, and track their listening history across individual user accounts. The design separates the source catalog from each user's personal collection and follows an MVC structure, written entirely by hand without GenAI. It was a two-person project for the same software engineering course, and I implemented the majority of the system, including the core library model, playlist management, user accounts, and play tracking. The full class design is captured in a UML class diagram.

  • Java
  • OOP
  • MVC

Baseball Performance Analytics

A data project with the Arizona Baseball Software Development Team, where I was project manager leading a group of eight. We built predictive performance models from TrackMan batted-ball data, monitoring the TrackMan system and collecting stadium measurements from Google Earth, which I processed. I wrote the Python analysis and engineered an "xHR" metric that estimates home run likelihood across more than 60 NCAA stadiums, and the scalable formula behind it was adopted program-wide to support data-driven player development. I also produced hitting reports for the coaching staff and led many of the team's working calls.

  • Python
  • Analytics
